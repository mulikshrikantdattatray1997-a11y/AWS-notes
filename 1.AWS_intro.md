

# 1. What is Cloud Computing?

## Definition (from your PPT)

Cloud computing is internet-based computing where remote servers provide:

* Data storage
* Processing power
* Access to services and applications

---

## Simple Explanation

Instead of:

* Storing data on your laptop
* Running applications on your own system

You use:

* Servers on the internet (called “cloud”)

---

## Real-life Examples

* **Google Drive**: Your files are stored on Google servers, not your device
* **Netflix**: Movies are streamed from cloud servers
* **Gmail**: Emails are stored in the cloud

---

## “Host of services contains”

From your slide:

* Networks
* Servers
* Storage
* Applications

Example in AWS:

* EC2 → servers
* S3 → storage
* RDS → database
* VPC → networking

---

## Key Point

> Services are available with minimal management and little/no provider interaction

Meaning:

* You don’t manage hardware
* Cloud provider handles infrastructure

---

## Pros

* No need to buy hardware
* Accessible from anywhere
* Easy to scale
* Cost efficient

## Cons

* Requires internet
* Security depends on configuration
* Vendor lock-in possible

---

# 2. Advantages of Cloud Computing

## 1. High Availability (99.9%)

System is almost always running.

Example:

* Google or AWS rarely go down

Why:

* Multiple backup servers are used

---

## 2. Fault Tolerance

If one server fails, another takes over.

Example:

* E-commerce sites continue working even if one server crashes

---

## 3. Scalability

You can increase or decrease resources as needed.

Example:

* During sales, apps increase server capacity
* After traffic reduces, they scale down

---

## 4. Elasticity

Automatic scaling based on demand.

Example:

* Sudden traffic spike → system automatically adds resources

Difference:

* Scalability = manual
* Elasticity = automatic

---

## 5. Lower Latency

Fast response time because servers are located globally.

Example:

* YouTube loads faster because it uses nearby servers

---

## 6. Pay-as-you-go Model

You only pay for what you use.

Example:

* Similar to electricity bill

---

---

### Cons

### Pros

* Low cost

### Cons

---

## 3. Community Cloud

### Meaning

Shared among specific organizations.

### From PPT

* Accessible to a group
* Similar to public cloud but limited access

### Examples

* Healthcare systems

### Providers

* IBM Cloud for Financial Services

### Pros

* Shared cost

### Cons

* Limited flexibility

Combination of private and public cloud.

### From PPT

* Total control

### Pros

* Flexible
* Secure + scalable

### Cons

* Complex setup
* Integration challenges

# Cloud Computing Service Models

Cloud services are divided into 3 main categories:

1. **IaaS (Infrastructure as a Service)**
2. **PaaS (Platform as a Service)**
3. **SaaS (Software as a Service)**

---

# First understand this foundation (VERY IMPORTANT)

Your second slide shows:

### Layers of a system:

* Applications
* Data
* Runtime
* Middleware
* Operating System
* Virtualization
* Servers
* Storage
* Networking

---

## Key idea

👉 Who manages what?

| Model      | You Manage | Cloud Provider Manages |
| ---------- | ---------- | ---------------------- |
| On-Premise | Everything | Nothing                |
| IaaS       | App → OS   | Infra                  |
| PaaS       | App + Data | Everything else        |
| SaaS       | Nothing    | Everything             |

---

# 1. IaaS (Infrastructure as a Service)

## Definition (from your PPT)

* Provides **virtualized computing resources over the internet**
* You rent:

  * Servers
  * Storage
  * Networking
* Pay-as-you-go

---

## What you manage

* Applications
* Data
* Runtime
* Middleware
* Operating System

## What cloud manages

* Virtualization
* Servers
* Storage
* Networking

---

## Simple Understanding

👉 You get a **virtual machine (like a remote computer)**
👉 You install everything yourself

---

## Real Example

* AWS EC2
* Google Compute Engine
* Azure Virtual Machine

---

## Practical Example

You launch an EC2 instance:

* You choose OS (Linux/Windows)
* You install Node.js
* You deploy your backend

👉 AWS only gives you the machine
👉 Everything inside = your responsibility

---

## Pros

* Full control
* Flexible
* Good for custom setups

## Cons

* Requires DevOps knowledge
* Maintenance effort
* More responsibility

---

## When to use

* Custom backend systems
* Legacy applications
* Full control required

---

# 2. PaaS (Platform as a Service)

## Definition (from PPT)

* Provides a platform to:

  * Build
  * Deploy
  * Manage applications
* Without managing infrastructure

---

## What you manage

* Applications
* Data

## What cloud manages

* Runtime
* Middleware
* OS
* Infrastructure

---

## Simple Understanding

👉 You focus only on **writing code**
👉 Everything else is handled

---

## Real Examples

* AWS Elastic Beanstalk
* Google App Engine
* Azure App Service

---

## Practical Example

You upload your Node app:

* No server setup
* No OS management
* No scaling setup

👉 Platform automatically handles:

* Deployment
* Scaling
* Load balancing

---

## Pros

* Fast development
* No infra management
* Auto scaling

## Cons

* Less control
* Platform restrictions
* Vendor lock-in

---

## When to use

* Startups
* MVPs
* Rapid development

---

# 3. SaaS (Software as a Service)

## Definition (from PPT)

* Third-party providers deliver software over the internet
* Subscription-based
* Also called “on-demand software”

---

## What you manage

👉 Nothing (just usage)

## What cloud manages

👉 Everything:

* Application
* Data
* Infrastructure

---

## Simple Understanding

👉 You just **use the software**
👉 No setup, no maintenance

---

## Real Examples

* Google Docs
* Gmail
* Microsoft 365

---

## Practical Example

Using Gmail:

* You don’t install server
* You don’t manage database
* You don’t handle scaling

👉 Everything handled by provider

---

## Pros

* No setup required
* Accessible anywhere
* Easy to use

## Cons

* No control
* Limited customization
* Data dependency on provider

---

## When to use

* End-user applications
* Business tools (email, CRM, etc.)

---

# Key Comparison (VERY IMPORTANT)

| Feature      | IaaS | PaaS   | SaaS |
| ------------ | ---- | ------ | ---- |
| Control      | High | Medium | Low  |
| Flexibility  | High | Medium | Low  |
| Ease of Use  | Low  | Medium | High |
| Setup Effort | High | Medium | None |

---

# Real-life Analogy

Think of food:

| Model | Example                               |
| ----- | ------------------------------------- |
| IaaS  | Raw ingredients → you cook everything |
| PaaS  | Semi-cooked meal → you just prepare   |
| SaaS  | Restaurant → just eat                 |

---

# Important Insight (Core Understanding)

* Moving from **IaaS → SaaS**

  * Control decreases
  * Convenience increases

---

# How AWS fits into this

* IaaS → EC2
* PaaS → Elastic Beanstalk
* SaaS → WorkMail, etc.

---

# Final Summary

* IaaS → You manage OS and above
* PaaS → You manage only app + data
* SaaS → You manage nothing

# 1. Basics of AWS

* Anyone can create an account and use AWS
* Resources are shared across users (logically isolated)

Example:

* You and Netflix both use AWS, but your data is isolated

AWS was one of the first to:

* Provide servers on rent (EC2)
* Provide storage on demand (S3)

It solved a major problem:

> Companies no longer needed to build physical data centers

---

## What happened in 2006?

Before AWS:

* Companies had to:

  * Buy servers
  * Set up data centers
  * Maintain hardware

After AWS:

* You could launch a server in minutes

---

AWS services are accessed via:

* Internet (browser, API, CLI)

Example:

* You don’t install AWS
* You access it online

---

* Data is replicated across multiple locations

* Increase or decrease resources anytime

* Pay only for usage

* Run your API

Now your backend is live on the internet.

# 2. What is EC2?

## Definition

**EC2 (Elastic Compute Cloud)** is an AWS service that provides **resizable compute capacity (virtual servers)**.

---

## Break the name

* **Elastic** → can scale up/down
* **Compute** → processing power
* **Cloud** → on internet

---

## Key Idea

EC2 allows you to:

> Rent virtual servers instead of buying physical servers

---

# 3. How EC2 Works (Step-by-step)

1. You request a server
2. AWS creates a virtual machine
3. You connect to it (SSH/RDP)
4. You install software
5. You run your application

---

# 4. Important Components of EC2

---

👉 Pre-configured OS template

Examples:

* Ubuntu
* Amazon Linux
* Windows Server

---

## 2. Instance Type

👉 Defines power of machine

Examples:

* t2.micro → small (free tier)
* m5.large → bigger

---

## 3. Storage (EBS)

👉 Disk attached to instance

* Like your laptop hard disk

---

## 4. Key Pair

👉 Used for login

* Like password (but more secure)

---

## 5. Security Group

👉 Firewall for your instance

Controls:

* Who can access your server
* Which ports are open

Example:

* Port 22 → SSH
* Port 80 → HTTP

# 2. Region (Most Important Concept)

## Definition

A **Region** = a **geographic area** where AWS has data centers.

---

## Simple Example

* Mumbai → `ap-south-1`
* Virginia → `us-east-1`

---

## Key Points

* Each region has **multiple data centers**
* Completely **isolated from other regions**
* Used for:

  * Data residency
  * Disaster recovery
  * Latency control

---

## Naming Pattern

```
ap-south-1
│   │     │
│   │     └── number
│   └──────── region
└──────────── continent
```

---

## Real-world Understanding

If your users are in India → choose **Mumbai region**

# 1. Cloud Computing Characteristics (Naming Conventions)

These are the **official 5 characteristics of cloud computing** (used by AWS + NIST)

---

## 1. On-Demand Self Service

### Meaning

You can create resources **instantly without human interaction**

### Example

* Launch EC2 in 2 minutes
* Create S3 bucket anytime

👉 No need to contact AWS support

---

## 2. Broad Network Access

### Meaning

Cloud services are available **over the internet from anywhere**

### Example

* Access AWS from laptop, mobile, tablet
* Open Gmail from anywhere

👉 Only internet required

---

## 3. Resource Pooling

### Meaning

Cloud provider shares resources among multiple users

👉 But data is **logically isolated**

### Example

* Same AWS server → used by many customers
* Your data remains private

---

## 4. Rapid Elasticity

### Meaning

Resources can be **scaled up/down instantly**

### Example

* Traffic spike → auto-scale EC2
* Traffic drops → reduce servers

👉 Looks “infinite” to user

---

## 5. Measured Service (Pay-as-you-go)

### Meaning

Usage is **monitored and billed based on consumption**

### Example

* EC2 → charged per hour
* S3 → charged per GB

👉 Similar to electricity bill

---

## Easy Way to Remember

```id="cloud5"
On-demand + Network + Pooling + Elasticity + Measured
```

---

## Quick Summary Table

| Characteristic   | Meaning                    |
| ---------------- | -------------------------- |
| On-demand        | Create resources instantly |
| Broad Access     | Available via internet     |
| Resource Pooling | Shared infra               |
| Elasticity       | Auto scaling               |
| Measured Service | Pay per use                |

# 2. AWS Shared Responsibility Model

👉 This is **VERY IMPORTANT (frequently asked in interviews)**

---

## Core Idea

> AWS handles **cloud security**
> You handle **security in the cloud**

---

## Simple Understanding

| Responsibility       | Who Handles |
| -------------------- | ----------- |
| Physical servers     | AWS         |
| Data center security | AWS         |
| Networking           | AWS         |
| OS, Apps, Data       | YOU         |

---

## Breakdown (VERY IMPORTANT)

---

## AWS Responsibility (Security OF the Cloud)

AWS manages:

* Data centers
* Physical servers
* Networking hardware
* Storage devices
* Virtualization layer

👉 You don’t worry about hardware

---

## Your Responsibility (Security IN the Cloud)

You manage:

* Operating System (EC2)
* Applications
* Data
* User access (IAM)
* Firewall rules (Security Groups)

---

## Example (EC2 Case)

If your server gets hacked:

👉 Possible reasons:

* Weak password
* Open port (22 to world)
* No updates

👉 This is **YOUR responsibility**, not AWS

---

## Important Difference by Service Type

---

### 1. IaaS (EC2)

👉 You manage more:

* OS
* Apps
* Security

---

### 2. PaaS (Elastic Beanstalk)

👉 AWS manages more:

* OS
* Runtime

👉 You manage:

* App + Data

---

### 3. SaaS (Gmail, AWS WorkMail)

👉 AWS manages almost everything

👉 You just use it

---

## Visual Understanding

```id="shared"
SaaS → Least responsibility  
PaaS → Medium  
IaaS → Maximum responsibility
```

---

## Real-Life Analogy

Think of renting:

| Model       | Responsibility         |
| ----------- | ---------------------- |
| Own house   | Everything yours       |
| Rental flat | Owner handles building |
| Hotel       | Almost nothing yours   |

---

## Common Mistakes (Important)

❌ “AWS will secure everything” → WRONG
❌ Leaving ports open
❌ Not using IAM properly

---

## Interview Answer (Perfect Line)

If asked:

👉 *“What is shared responsibility model?”*

Answer:

> AWS secures the infrastructure (hardware, networking),
> while customers are responsible for securing their applications, OS, and data.

# 🔹 EC2 (Elastic Compute Cloud)

👉 **What it is:**
Virtual server in AWS (like a remote computer)

👉 **Used for:**

* Running backend (Node, Java, Python)
* Hosting websites
* APIs

👉 **Key Idea:**
You rent a machine → install everything yourself

---

## Important EC2 Terms

* **Instance** → Virtual server
* **AMI** → OS template (Ubuntu, Windows)
* **Instance Type** → CPU/RAM (t2.micro, etc.)
* **EBS** → Storage (hard disk)
* **Key Pair** → Login access (like password)
* **Security Group** → Firewall (controls ports like 22, 80)

---

# 🔹 S3 (Simple Storage Service)

👉 **What it is:**
Cloud storage for files (images, videos, backups)

👉 **Used for:**

* Storing files
* Static websites
* Backups

👉 **Key Idea:**
Unlimited storage, highly durable

---

## Important S3 Terms

* **Bucket** → Container for files
* **Object** → File stored in S3
* **Key** → File name/path
* **Versioning** → Keeps file history
* **ACL / Bucket Policy** → Access control
* **Storage Classes** → Cost tiers (Standard, Glacier)

---

# 🔹 EC2 vs S3 (Simple Difference)

| Feature | EC2          | S3          |
| ------- | ------------ | ----------- |
| Type    | Compute      | Storage     |
| Use     | Run apps     | Store files |
| Control | Full control | Managed     |

---

# 🔹 Related Core AWS Terms

* **IAM** → User & permission control
* **VPC** → Private network in AWS
* **RDS** → Managed database
* **CloudWatch** → Monitoring/logs
* **ELB (Load Balancer)** → Distributes traffic
* **CloudFront** → CDN (fast delivery)
